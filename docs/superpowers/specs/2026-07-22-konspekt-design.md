# Конспект — дизайн MVP

**Дата:** 2026-07-22
**Статус:** утверждён (дизайн), ожидает плана реализации

## Идея

Телеграм-бот, который борется с «цифровым хламом» — сохранёнками, которые
никогда не открываются. Кидаешь боту статью, видео или документ — бот делает
**конспект** (суть + тезисы + теги) и присылает его прямо в чат сообщениями.
Ключевая механика против «кладбища сохранёнок»: контент не копится, а сразу
превращается в короткий читаемый конспект, который можно потребить за минуту
и разобрать кнопками триажа.

## Цель и ценность

- **Для кого:** лично для владельца — инструмент ежедневного использования.
- **Боль:** «сохранил на потом и не открыл»; длинные видео и статьи, которые
  жаль смотреть целиком.
- **Решение:** мгновенная выжимка сути в Telegram + лёгкий триаж.

## Область MVP

Точка входа — только Telegram-бот. Веб-интерфейса нет.

Обрабатываем три типа входа:

1. **Статьи** — ссылка на веб-страницу → извлечение читаемого текста.
2. **YouTube** — ссылка на видео → транскрипт → конспект.
3. **Документы** — присланный в бот файл (PDF, DOCX) → текст → конспект.

Всё остальное (twitter, подкасты, произвольные видео без транскрипта) — вне MVP;
для неподдерживаемого входа бот честно сообщает об этом.

## Стек

| Слой | Выбор | Обоснование |
|------|-------|-------------|
| Язык | TypeScript (Node 25) | Типобезопасность, экосистема |
| Бот | grammY (long-polling) | TS-first, не нужен публичный URL для разработки |
| БД | Prisma + SQLite | Ноль инфраструктуры для MVP; миграция на Postgres тривиальна |
| LLM | OpenRouter (OpenAI-совместимый) | Один ключ, выбор модели через env |
| Статьи | `@extractus/article-extractor` | Readability под капотом |
| YouTube | `youtube-transcript` | Получение субтитров/транскрипта |
| PDF | `pdf-parse` | Извлечение текста из PDF |
| DOCX | `mammoth` | Извлечение текста из DOCX |

Модель по умолчанию задаётся в `.env` (например `anthropic/claude-3.5-haiku`) —
дёшево и быстро для конспектов. Ключ OpenRouter и токен бота — только в `.env`,
в git не попадают.

## Поток данных

```
Пользователь шлёт: ссылку / YouTube / файл
        │
        ▼
  Роутинг по типу входа
        │
   ┌────┴─────┬───────────┐
   ▼          ▼           ▼
Статья    YouTube     Документ
(extract) (transcript) (pdf/docx)
   └────┬─────┴───────────┘
        ▼
  Нормализованный текст + метаданные (title, sourceType, url)
        │
        ▼
  OpenRouter: промпт → { summary, keyPoints[], tags[] }
        │
        ▼
  Сохранение Item в БД (status=inbox)
        │
        ▼
  Бот шлёт карточку-конспект в чат
  [📖 Подробнее] [✅ Готово] [🗑 Удалить]
```

### Формат сообщения-конспекта

- **Заголовок** (жирным) + иконка типа источника.
- **Суть** — 2–3 предложения.
- **Тезисы** — маркированный список ключевых пунктов.
- **Теги** — 3–5 штук.
- Инлайн-кнопки триажа.

Telegram-лимит сообщения 4096 символов: длинный «Подробнее» бьётся на несколько
сообщений по границам абзацев.

## Триаж

Кнопки под конспектом меняют `status` у `Item`:

- **📖 Подробнее** → досылает развёрнутый разбор отдельными сообщениями
  (не меняет статус; можно нажать повторно).
- **✅ Готово** → `status = done` (разобрано, конспект прочитан).
- **🗑 Удалить** → `status = deleted` (мягкое удаление, не хард-делит).

Obsidian **не** является приёмником сохранённого контента — весь выход идёт
сообщениями в Telegram. Obsidian используется только для документации проекта.

## Модель данных (Prisma)

```prisma
model Item {
  id                String   @id @default(cuid())
  sourceType        SourceType
  url               String?          // для статей/видео; null для присланных файлов
  title             String
  summary           String
  keyPoints         Json             // string[]
  tags              Json             // string[]
  rawText           String?          // извлечённый текст (для «Подробнее»/дебага)
  status            Status   @default(inbox)
  telegramUserId    String
  telegramChatId    String
  createdAt         DateTime @default(now())

  @@index([telegramUserId, status])
}

enum SourceType { ARTICLE YOUTUBE PDF DOCX }
enum Status { inbox done deleted }
```

Дубликаты URL детектируются по паре (`telegramUserId`, `url`) в статусах
`inbox`/`done`: бот сообщает «уже есть» и не создаёт повторный конспект.

## Обработка ошибок и крайние случаи

| Ситуация | Поведение |
|----------|-----------|
| Fetch/парсинг статьи не удался (paywall, JS, 404) | Фолбэк на og:title/description; если и их нет — сообщение «не смог извлечь текст» |
| У YouTube-видео нет транскрипта | Сообщение «нет субтитров, конспект невозможен» |
| Файл не PDF/DOCX или битый | Сообщение о неподдерживаемом/повреждённом файле |
| Файл слишком большой | Проверка лимита размера до скачивания; отказ с понятным текстом |
| LLM (OpenRouter) упал | Один ретрай; при повторной ошибке — сообщение об ошибке, Item не создаётся |
| Текст длиннее контекста модели | Обрезка до безопасного лимита символов перед отправкой в LLM |
| Не-URL и не-файл сообщение | Мягкая подсказка «пришли ссылку или файл» |
| Дубликат URL | «Уже есть», ссылка на прошлый конспект |

## Структура кода (модули с чёткими границами)

- `src/bot/` — grammY: регистрация хендлеров, роутинг входа, рендер карточки,
  колбэки кнопок. Зависит от сервисов, не знает их внутренностей.
- `src/extractors/` — по одному модулю на тип: `article.ts`, `youtube.ts`,
  `pdf.ts`, `docx.ts`. Общий интерфейс `extract(input) → ExtractedContent`.
- `src/summarizer/` — клиент OpenRouter + промпт + парсинг структурированного
  ответа в `{ summary, keyPoints, tags }`.
- `src/storage/` — Prisma-репозиторий `Item` (create, findByUrl, updateStatus).
- `src/format/` — сборка текста сообщения и разбиение по лимиту 4096.
- `src/config.ts` — чтение и валидация env.

Каждый модуль тестируется изолированно; бот-слой — тонкий оркестратор.

## Тестирование

- **Юнит:** роутинг URL/типа входа; каждый экстрактор на фикстурах
  (сохранённый HTML / пример транскрипта / маленькие PDF и DOCX);
  форматирование сообщения и разбиение по 4096; генерация промпта.
- **Моки:** OpenRouter-клиент и сетевой `fetch` замоканы — тесты детерминированы,
  без реальных запросов и трат токенов.
- **Хендлеры бота:** через grammY testing utilities с фейковым контекстом.

## Что вне MVP (YAGNI)

- Веб-дашборд и поиск по архиву.
- Экспорт контента в Obsidian.
- Ежедневный дайджест / напоминания / TTL-«банкротство».
- Мультиязычные конспекты, кастомные промпты на пользователя.
- Postgres, деплой, вебхуки (пока long-polling локально).
