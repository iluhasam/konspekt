# ADR-0001 — Стек и архитектура

## Статус

Принято — 2026-07-22.

## Контекст

Нужен телеграм-бот, который принимает статью/видео/документ, делает конспект
через LLM и присылает его сообщениями. Личный проект ежедневного использования,
приоритет — быстро дойти до «я им пользуюсь», при этом сохранить учебную
ценность (Prisma-трек) и держать чистые границы модулей.

## Решения

### Бот: grammY на long-polling
TypeScript-first библиотека. Long-polling не требует публичного URL —
запускается локально без вебхуков и туннелей. Вебхуки/деплой отложены.

### БД: Prisma + SQLite
Ноль инфраструктуры для MVP. Переход на Postgres позже тривиален (сменить
datasource). Сохраняем практику Prisma.

### LLM: OpenRouter
OpenAI-совместимый endpoint (`https://openrouter.ai/api/v1`). Один ключ, выбор
модели через env — можно дёшево гонять на быстрой модели и менять без кода.

### Извлечение: по модулю на тип входа
`@extractus/article-extractor` (статьи), `youtube-transcript` (видео),
`pdf-parse` (PDF), `mammoth` (DOCX). Общий интерфейс `extract → ExtractedContent`,
чтобы бот-слой не знал внутренностей.

### Хранилище конспектов — БД, не Obsidian
Весь пользовательский выход идёт в Telegram. Obsidian зарезервирован под
документацию проекта. Это удерживает единственный канал потребления (чат) и не
плодит источников правды.

## Последствия

- **Плюсы:** минимум инфраструктуры, быстрый локальный старт, чистые границы,
  дешёвая смена LLM-модели.
- **Минусы / долги:** SQLite и long-polling не для продакшена — при деплое
  понадобятся Postgres и вебхуки; это осознанный отложенный долг.
